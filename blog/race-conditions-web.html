Qu'est-ce qu'une race condition ?

Une race condition survient quand le comportement d'un système dépend de l'ordre d'exécution de plusieurs opérations concurrentes. En web, cela se produit quand plusieurs requêtes modifient le même état simultanément.

Pattern TOCTOU : Time-Of-Check to Time-Of-Use. Le système vérifie une condition (ex: solde suffisant) puis effectue l'action (ex: débit). Entre les deux, un attaquant peut modifier l'état.

Scénarios d'exploitation

1. Double-spending

Dépenser le même solde plusieurs fois :

# Pseudo-code vulnérable
def transfer(user, amount, to):
    if user.balance >= amount:    # CHECK
        time.sleep(0.1)           # Window!
        user.balance -= amount    # USE
        to.balance += amount
        return "Success"
    return "Insufficient funds"

2. Rate limit bypass

Si le rate limit est vérifié avant d'incrémenter le compteur, plusieurs requêtes simultanées passent toutes.

3. Coupon reuse

Utiliser le même code promo plusieurs fois avant qu'il soit marqué comme utilisé.

HTTP/2 Single-packet attack

HTTP/2 permet d'envoyer plusieurs requêtes dans un seul paquet TCP, garantissant qu'elles arrivent simultanément :

import httpx

async def race_attack():
    async with httpx.AsyncClient(http2=True) as client:
        # Préparer les requêtes
        requests = [
            client.post('/transfer', data={'amount': 100})
            for _ in range(10)
        ]
        # Les envoyer simultanément
        responses = await asyncio.gather(*requests)
        return responses

Conclusion

Les race conditions sont souvent sous-estimées car difficiles à reproduire en test manuel. Les outils comme Turbo Intruder et les techniques HTTP/2 les rendent exploitables de manière fiable.
