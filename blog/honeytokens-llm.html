Utiliser les modèles de langage pour générer des données leurres réalistes et indétectables, puis détecter leur exfiltration.

Qu'est-ce qu'un honeytoken ?

Un honeytoken est une donnée fictive placée stratégiquement dans un système. Si elle est accédée ou exfiltrée, c'est un signe certain d'intrusion car aucun utilisateur légitime n'a de raison de la toucher.

L'avantage des honeytokens : Contrairement aux IDS classiques qui génèrent des faux positifs, un honeytoken déclenché est une preuve certaine d'activité malveillante. Pas de bruit, que du signal.

Exemples classiques

  • Fichiers passwords.txt avec des credentials monitorés
  • Comptes utilisateurs inactifs avec alertes sur connexion
  • URLs uniques dans des documents sensibles
  • Faux records dans des bases de données

Le problème : réalisme

Les honeytokens traditionnels sont souvent détectables par un attaquant expérimenté. Un fichier passwords.txt trop propre ou des données trop régulières éveillent les soupçons.

Solution : génération par LLM

Les LLMs peuvent générer des données qui imitent parfaitement le style et le format des vraies données de l'entreprise.

Génération de faux credentials

import ollama

def generate_fake_creds(company_name, num_entries=10):
    prompt = f"""Generate {num_entries} realistic fake employee credentials 
    for {company_name}. Include:
    - Email (firstname.lastname@company.com format)
    - Password (mix of patterns people actually use)
    - Department
    - Last login date
    
    Make them look like real leaked data, not too clean."""
    
    response = ollama.generate(model='llama3.2', prompt=prompt)
    return response['response']

# Générer des honeytokens
fake_data = generate_fake_creds("Acme Corp", 20)
print(fake_data)

Documents leurres

def generate_honey_document(doc_type, company_context):
    prompts = {
        "financial": "Generate a realistic quarterly financial report...",
        "hr": "Generate employee performance review notes...",
        "technical": "Generate API documentation with fake endpoints..."
    }
    
    response = ollama.generate(
        model='llama3.2', 
        prompt=prompts[doc_type] + f"\nCompany context: {company_context}"
    )
    return response['response']

Détection de l'exfiltration

Une fois les honeytokens placés, il faut détecter leur utilisation :

  • Canary tokens — URLs uniques qui alertent quand accédées
  • Monitored credentials — Comptes qui alertent sur tentative de connexion
  • DNS exfil detection — Domaines uniques dans les documents

Intégration Canarytokens

# Créer un canary token (exemple avec l'API)
import requests

def create_canary_url(memo):
    response = requests.post('https://canarytokens.org/generate', data={
        'type': 'web',
        'email': 'security@company.com',
        'memo': memo
    })
    return response.json()['token_url']

# Insérer dans un document honeytoken
honey_url = create_canary_url("Financial Report Q4 accessed")
document = f"Confidential data available at: {honey_url}"

Architecture complète

  1. Génération — LLM crée des données réalistes
  2. Placement — Déploiement stratégique (shares, databases, documents)
  3. Monitoring — Canary tokens + intégration SIEM
  4. Alerte — Notification immédiate + enrichissement contextuel

Conclusion

Les LLMs ouvrent de nouvelles possibilités pour la deception technology. Des honeytokens indistinguables des vraies données rendent la détection d'intrusion plus fiable et moins sujette aux faux positifs.
